## Анализ требований — функционал корзины

### Обнаруженные противоречия
- Пункты 2 и 9 противоречат друг другу.  
- Пункт о смене рекламы не относится к корзине.  
- Не определено поведение при изменении цены во время оформления заказа.

### Исправление артефактов
- Убрать противоречивые пункты.  
- Перенести рекламный функционал в другой раздел.  
- Добавить правило обновления цены: если заказ не оформлен — обновить автоматически; если оформляется — уведомить пользователя.

### Конечный вариант требований (T3)
1. Пользователь может добавить в корзину от 1 до 10 единиц одного товара.  
2. Общий лимит корзины — 20 условных едниц, но не более 5 различных товаров
3. После добавления товара система показывает сообщение «Товар добавлен».  
4. При удалении товара он исчезает из списка корзины.  
5. Если количество товара уменьшено до 0 — товар удаляется.  
6. На странице корзины отображаются название, количество, цена за единицу и итоговая сумма.  
7. При изменении цены в каталоге система обновляет цену в корзине, если заказ не оформлен.

## Архитектура PUSH‑уведомлений

### Цель системы

Обеспечить доставку уведомлений пользователям мобильного приложения через масштабируемую микросервисную архитектуру.

### Архитектурные компоненты

• Frontend (мобильное приложение) — получение уведомлений через FCM/APNS.

• Notification Service — формирование и отправка уведомлений.

• Event Bus (Kafka/RabbitMQ) — асинхронная передача событий.

• Order Service — события о заказах (отмена, задержка).

• Marketing Service — рекламные рассылки.

• User Service — токены устройств и настройки уведомлений.

• Scheduler Service — проверка «зависших» корзин.


### Пример потока данных

1. Order Service создаёт событие order_cancelled. 

2. Событие отправляется в Event Bus. 

3. Notification Service формирует уведомление и вызывает FCM/APNS.

4. Уведомление доставляется пользователю.



### Технологический стек

| Компонент | Технология | Назначение |
|------------|-------------|-------------|
| Notification Service | Node.js / Python | Формирование уведомлений |
| Event Bus | Kafka / RabbitMQ | Асинхронная доставка событий |
| Database | PostgreSQL / Redis | Хранение токенов и шаблонов |
| Mobile App | FCM / APNS | Получение уведомлений |
